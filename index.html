Route C · the split

Generators cannot predict. Predictors cannot generate.

No single model does both — so no single model can be checked on both.

Our own wall · measured

A from-scratch 100M model collapses to the mean

Predict-the-change → static video. Predict-the-state → incoherent video. The missing ingredient was never the parameterization. It was the prior.

Insight · click a card for the inspiration

Pretraining was given to the encoder. Nobody gave it to the generator.

Insight 01 · the space

Robustness belongs to the space, not the trainer

Frozen features are smooth: a slightly wrong prediction is still a valid state. That holds for any model running there.

So — a generator need not be trained from scratch to inherit drift-robustness. It only has to be moved in.

Insight 02 · the prior

A video prior is knowledge about change, not about channels

Objects persist, motion is smooth, scenes stay consistent. None of that is welded to one VAE's channels.

So — swap the input/output projections, keep the blocks, and the knowledge should survive the move. This is the falsifiable core.

Insight 03 · the two jobs

Generation and prediction differ in compression, not in space

One space, one mechanism — and a single dial: how tightly we compress it.

So — tight latents for generation, high-resolution latents for readout. Same surgery, one knob. Slide 8 makes this a testable prediction.

Trained part 2 of 2 · ★ the mechanism

Keep the body. Replace the sockets.

Baseline · faithful compression

Latent 7×7 — the model's own patchifier is discarded

The latent is already compact, so the pretrained input conv no longer fits. The 30 blocks stay: that is where the prior lives.

Pre-registered prediction · the falsifier

Transfer more prior → the gap must widen

Toggle above: shape the latent like the model's own input, and even the patchifier and output projection survive. If the advantage does not grow, then "amount of prior" is not the mechanism — and we shrink the claim, publicly.

Two edits · generator → world model

Make it causal. Make every frame its own noise level.

Training objective: regress the velocity from noise to data. The only variable between the two headline arms is the initialization.

New evaluation setting

One rollout, forced through two authoritative pipelines.

Existing protocols score a generator or a predictor. We hold the model fixed and score the same sample on both — making "broad capability" a measurement, not a story.

Risks · click for the contingency

Every risk has an exit, decided before the run.

+
Resolution

7×7 latents may be too coarse for segmentation

Competing predictors read out far finer.

Exit — split the compression dial: tight latents for generation, high-resolution latents for readout. The prior-aligned codec already gives this for free.

+
Ceiling

VBench scores the decoder, not the world model

A weak renderer can hide a strong model.

Exit — always evaluate a ground-truth-decoded control set to expose the ceiling. If the decoder binds, generation becomes the secondary claim and prediction carries the paper.

+
Mechanism

Maybe it is the architecture, not the prior

Perhaps the block design just happens to be good.

Exit — the scratch arm is architecturally identical, so this is controlled. The prior-aligned test pushes harder: more prior must mean a bigger gap, or we shrink the claim.

+
Infrastructure

The cluster is full; the flagship cannot start

Wall-clock, not GPU budget, is the binding constraint.

Exit — the science was front-loaded onto cheap runs: the decisive gate cost two GPUs. Scale buys strength, not validity.

The prior survives the transplant.
